## 2) ЖЁСТКИЕ ОГРАНИЧЕНИЯ (обязательно)
1. **Запрещено использовать индексы** (price index / index series).  
2. **Запрещено ссылаться на индексы** как на источник данных для транзакций.  
3. В отчётном периоде использовать **только метрику: Total Transactions (count)** для **жилой** недвижимости.  
4. **Запрещено выдумывать**: цифры, диапазоны, “оценки”, “примерно”, восстановление пропусков.  
5. Если официальные данные **не содержат** разбивку по типам жилья — ты обязан:
   - либо найти другой **государственный официальный источник**, где есть разбивка,  
   - либо честно указать, что разбивки по типам нет, и **не строить** графики по типам (но построить общий график “Overall”, если он официально подтверждён).

## 3) ДОПУСТИМЫЕ ИСТОЧНИКИ (только официальные)
Разрешены **только** источники уровня:
- **Dubai Land Department (DLD)** (включая официальные порталы/дашборды/публикации DLD)
- Другие **государственные источники ОАЭ**, если они дают **официальные** цифры транзакций по ОАЭ (например, федеральные/эмиратские статистические органы, земельные департаменты, гос-порталы статистики)

**Запрещено**: брокеры, порталы объявлений, частные ресёрчи (Knight Frank / CBRE / JLL и т.п.) как источник “официальных” чисел (их можно упоминать только как комментарий, но **не** как источник цифр).

## 4) ТРЕБОВАНИЯ К ДОКАЗАТЕЛЬСТВАМ (Evidence Pack)
Для **каждой цифры** в итоговой таблице обязателен источник:

**Source Register** (таблица источников) — для каждого источника:
- Source_ID (S1, S2, …)
- Организация (например, DLD)
- Название страницы/отчёта/дашборда
- URL
- Дата публикации/обновления (если есть)
- Метод доступа (таблица/дашборд/API/CSV/PDF)
- Поля/метрики, которые источник подтверждает
- Evidence_Location (страница/таблица/виджет/скрин/параметры фильтра)
- Примечания/ограничения

**Fact Table** (таблица фактов) — каждая строка данных должна иметь:
- Geography (Dubai / UAE)
- Period (YYYY-Q#)
- Property_Type (Apartment / Villa / Townhouse / Overall / …)
- Metric (Total Transactions)
- Value
- Unit (= count)
- Transaction_Type (если источник различает: Sales/Transfers/Buy/Sell; иначе “Total” строго по формулировке источника)
- Residential_Flag (= Residential подтверждено источником)
- Source_ID
- Evidence_Location (точное место в источнике + фильтры)

1. Найди официальные источники, где можно получить **квартальные** данные 2020–2025.  
2. Если данные доступны **помесячно**, разрешено агрегировать в квартал, но строго:
   - Quarterly Total = сумма месячных значений в квартале  
   - И обязательно помечать строки как **CALCULATED_FROM_OFFICIAL_MONTHLY**, с ссылками на месячные строки/таблицы.
3. Если данные доступны по годам — это **недостаточно** (кварталы обязательны). Ищи другой официальный источник или другой способ выгрузки (CSV/дашборд/API).
